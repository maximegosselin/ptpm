# Pull The Plug Modeling: Examples

This repository contains concrete examples where [Pull The Plug Modeling](https://maximegosselin.com/posts/pull-the-plug-modeling/)
proves beneficial for software design.

PTPM is a mental exercise: imagine your domain without electricity, just people, paper, and pens.
By removing technological assumptions, it surfaces the authentic domain beneath the layers of technical
design, and forces better questions before any implementation decisions are made.

## Examples

- [Collaborative Diagram Tool](diagram-collaboration-tool/): what happens when you ask an AI for
  solutions to a real-time collaboration problem, with and without PTPM.
